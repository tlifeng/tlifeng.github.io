隔离级别

Levi Tang 2020-12-20 13:22:26

Mysql和Postgres事务隔离级别

隔离级别概念

未提交读（READ UNCOMMITTED）

事务没有提交，别的事务也可以读到修改（脏读）。

提交读（READ COMMITTED）

除了mysql的大多关系数据库的默认级别都是提交读（pg, oracle等），a事务可以看到b事务的提交修改，所以a事务过程开始和结束查询的记录都可能不一样，为不可重复读。

可重复读（REPEATABLE READ）

a事务不能读到b事务提交的修改，所以a事务过程中读的数据都是一样的，为可重复读。b事务插入了新记录，a事务能查看到，存在幻读问题（innodb通过多版本控制和gap锁没有幻读）。

可串行化（SERIALIZABLE）

a事务开始时，读取一条数据会加锁，不允许b事物再进行修改。会有锁争用问题导致超时。

比较Mysql和Postgresql隔离级别区别

mysql查看和修改隔离级别命令

要修改全局隔离级别则把下面session换成global。

# 查看当前事务隔离级别
SELECT @@tx_isolation;

# 设置read uncommitted级别：
set session transaction isolation level read uncommitted;

# 设置read committed级别：
set session transaction isolation level read committed;

# 设置repeatable read级别：
set session transaction isolation level repeatable read;

# 设置serializable级别：
set session transaction isolation level serializable;

# 开始事务
begin;

# 提交事务
commit;

# 回滚
rollback;

postgre查看和修改隔离级别命令

# 查看默认隔离级别
show default_transaction_isolation;

# 查看当前隔离级别
show transaction_isolation;

# 只在事务中使用有效
# 设置事物中不同隔离级别：
set transaction isolation level read uncommitted;
set transaction isolation level read committed;
set transaction isolation level repeatable read;
set transaction isolation level serializable;

# 修改当前会话默认隔离级别
set default_transaction_isolation='read uncommitted';
set default_transaction_isolation='read committed';
set default_transaction_isolation='repeatable read';
set default_transaction_isolation='serializable';

所以提交读为不可重复读，因为a事务中因为能读到b事务的提交修改，导致两次查询code结果不一致。现在我们在b事物中插入一条新记录，看看a是否会看到（幻读）。

# b事务
> BEGIN;
Query OK, 0 rows affected (0.00 sec)

# ...表示省略插入的值
> INSERT INTO `position` VALUES (27699, ...);
Query OK, 1 row affected (0.00 sec)

> COMMIT;
Query OK, 0 rows affected (0.00 sec)

a事务查看最后三位id, 存在插入的id，存在幻读现象

SELECT id FROM position ORDER BY id DESC LIMIT 3;
+-------+
| id    |
+-------+
| 27699 |
| 27698 |
| 27697 |
+-------+
3 rows in set (0.00 sec)

总结： 提交读没有脏读，有不可重复读和存在幻读

b事务

> BEGIN;
Query OK, 0 rows affected (0.00 sec)

> UPDATE position SET code = "aaa" WHERE id = 1278;
1205 - Lock wait timeout exceeded; try restarting transaction

可串行化时查看表一个范围时，其他事物无法修改。

# a事物
> SELECT * FROM position WHERE id BETWEEN 1200 AND 1300;
...
...
101 rows in set (0.06 sec)

# b事务
> UPDATE position SET code = "bbb" WHERE id = 1278;
1205 - Lock wait timeout exceeded; try restarting transaction

总结： 串行化可重复读，没有脏读，没有幻读
